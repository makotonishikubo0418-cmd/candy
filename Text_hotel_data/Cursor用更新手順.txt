■ 正本
・元データ: HP/Text_hotel_data/対象ホテル.txt
・HTML: HP/source/template_kagoshima-deliveryhealth-hotel.html
・店舗: HP/source/template_shop.html
・実行: HP/codex/scripts/candy-hotel.cmd publish --input "HP/Text_hotel_data/対象ホテル.txt"

■ 最重要
・各項目の件数は原則固定しない。対象Textに存在する完成ブロックを全件、記載順で反映する。
・通常記事scene、FAQ、料金、アクセス、周辺スポットは0件でもよい。0件なら空セクションを作らない。
・subtitleだけ、descriptionだけ、アクセスの一部だけなど、部分入力は推測せずSTOPする。
・店舗はtemplate_shop.htmlに存在する店舗を1件以上指定する。
・関連記事は実リンク設定まで予約ダミー8件を維持する。
・元データにないホテル情報、画像、URL、文章を推測しない。
・aaaaaaaa、placeholder、未入力指示が残るTextは公開しない。

【旧option: 0または1】
option / option_subtitle / option_descriptionの3項目を全部書く。
通常sceneには数えない。3項目の一部だけは不可。

【通常記事scene: 0件以上】
scene（h2）
見出し
subtitle_
小見出し
description_
本文

必要数だけ繰り返せる。店舗、FAQ、基本情報等の前後に置け、入力順を保持する。

【FAQ: 0件以上】
scene（h2 / よくあるご質問）
subtitle_
質問
description_
回答

subtitle_ / description_ の組を必要数だけ繰り返す。

【料金: 0件以上】
scene（h2）
料金情報
休憩 / 休憩等 / ショート / フリー / 宿泊 / 延長
値

存在する料金行だけを書く。行数は固定しない。
料金行の後にdescription_と補足文を1件だけ追加できる。

【アクセス: 0または1】
scene（h2）
アクセス情報
地図URL / 地図タイトル / subtitle_ / description_

4項目を全部書く。一部だけは不可。店舗の移動時間・交通費が未指定の場合、地図座標を近隣完成areaページの参照に使う。

【周辺スポット: 0件以上】
scene（h2）
ホテル名「周辺スポット」
subtitle_
施設名
description_
住所
TEL 任意
URL https://...

subtitle_ / description_ の組を必要数だけ繰り返す。
全項目の後にdescription_と注意文を1件だけ追加できる。

生成ルール
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
・旧option以外のh2は、表示順にscene1から連番にする。
・通常ブロックはsubtitle_N / description_Nとする。
・FAQと周辺スポットはsubtitle_N_M / description_N_Mとする。
・FAQ型の最終項目だけfaq-item bd_tb、それ以外はfaq-item bd_tとする。
・FAQが0件ならFAQPage JSON-LDを作らない。
・周辺スポットがあればItemListはスポット、なければ店舗を使う。
・店舗の移動時間・交通費はText指定を最優先し、未指定時だけ地図座標と近隣完成areaページから設定する。
・空欄、空コンテナ、重複ID、scene欠番、本文とJSON-LDの件数不一致を許可しない。
