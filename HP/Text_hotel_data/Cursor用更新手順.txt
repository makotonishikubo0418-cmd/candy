■ 前提
・更新対象のHTMLは @template_kagoshima-deliveryhealth-area.html をベースとする。
・テンプレート内の更新箇所はすべて「aaaaaaaaaaaaaaaaaaaa」でマークされている。
・@Text_data/[エリア名].txt の仕様に従い、HTML内の「aaaaaaaaaaaaaaaaaaaa」を該当する内容に置き換える。
・エリア名（例：荒田）とファイル名用スラッグ（例：arata）は .txt やファイル名から判断する。

■ 依頼時のプロンプト例
「@Text_data/[エリア名].txt の内容で、@template_kagoshima-deliveryhealth-area.html をベースに kagoshima-deliveryhealth-area-[スラッグ].html を更新して。実行して。」

※ 事前に確認だけする場合は「修正せずに意見だけ言って」等を追記する。「実行して」と明示した場合のみファイルを編集すること。

置換一覧（aaaaaaaaaaaaaaaaaaaa → .txt の該当項目）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
※ URL・パス内の aaaaaaaaaaaaaaaaaaaa は、原則として「ファイル名用スラッグ」（例：arata）に置換する。
※ 表示文言の aaaaaaaaaaaaaaaaaaaa は、.txt の「エリア名」や該当項目の文言に置換する。

【1】head 内
・meta description の content
  → .txt「基本情報」の description
・title のテキスト
  → .txt「基本情報」の title
・link canonical の href
  → .txt「基本情報」の canonical（※ ここで aaaaaaaaaaaaaaaaaaaa をスラッグに置換）
・og:title の content
  → .txt「基本情報」の title に合わせる
・og:url の content
  → .txt「基本情報」の canonical
・og:image の content
  → .txt「基本情報」の image（※ パスに aaaaaaaaaaaaaaaaaaaa があればスラッグに置換）
・og:description の content
  → .txt「基本情報」の description

【2】構造化データ（JSON-LD）
・BreadcrumbList の position 3 の name
  → .txt「page_title_h1・パンくずリスト」と同じ文言
・BreadcrumbList の position 3 の item
  → .txt「基本情報」の canonical
・ItemList の name
  → 「鹿児島[エリア名]に呼べる人気デリヘル店一覧」の形式（.txt の scene1 に合わせる）
・ItemList の numberOfItems
  → .txt の scene1 で指定された店舗数
・ItemList の itemListElement
  → .txt の scene1 で指定された店舗の順で、各店の name / telephone / url / description を設定（template_shop.html や既存エリアページを参照）

【3】メインコンテンツ（パンくず・見出し・本文・画像）
・bread_list 内の span のテキスト
  → .txt「page_title_h1・パンくずリスト」
・#img_1 の img の src
  → .txt「記事内容」の 1 枚目の写真の src（※ aaaaaaaaaaaaaaaaaaaa をスラッグに置換。.txt に area/ が無ければ直下パス）
・#img_1 の img の alt
  → 「鹿児島市[エリア名]で呼べるデリヘル」等、.txt に合わせる
・#page_title_h1 のテキスト
  → .txt「page_title_h1・パンくずリスト」と同じ文言（改行は <br class="spOnly"> を維持）
・#subtitle_h1 の内容
  → .txt の subtitle_h1（<改行> は <br> に）
・#description_h1 のテキスト
  → .txt の description_h1

【4】scene1（人気デリヘル店）
・#scene1 の見出し
  → .txt の scene1
・<ul class="campaign-list"> 内
  → .txt の scene1 で指定された店舗のみ、template_shop.html の該当ブロック（<!-- candy -->, <!-- loveel -->, <!-- danzuma -->, <!-- yokihi --> 等）～</li> をコピーし、移動時間・交通費を .txt の数値で上書き。順序は .txt に従う。
・#description_1 のテキスト
  → .txt の description_1

【5】画像2・scene2・scene3
・#img_2 の img の src / alt
  → .txt の 2 枚目の写真の src および「鹿児島「[エリア名]」について」等
・#scene2 のテキスト
  → .txt の scene2
・#subtitle_2 のテキスト
  → .txt の subtitle_2
・#description_2 のテキスト
  → .txt の description_2
・#scene3 のテキスト
  → .txt の scene3
・iframe の title
  → 「鹿児島市 [エリア名]の地図」
・iframe の src
  → 当該エリアの Google マップ embed URL（.txt に無ければ座標等で作成または要確認）
・info-table1 の「人口」「面積」「設置年月日」の各 td
  → .txt の scene3 直下の値（未記載は「—」）
・#description_3 のテキスト
  → .txt の description_3

【6】scene4（ホテル）
・#scene4 の見出し
  → .txt の scene4（※ .txt に他エリア名が残っていても正しいエリア名に直す）
・faq-item の数
  → .txt の subtitle_4_n / description_4_n の件数に合わせ、ブロックの追加・削除で調整
・各 #subtitle_4_n / #description_4_n
  → .txt の subtitle_4_n（施設名）、description_4_n（住所・TEL・URL）。リンクは target="_blank" rel="noopener noreferrer" を付与

【7】scene5（待ち合わせ・周辺スポット）
・#scene5 の見出し
  → .txt の scene5（※ 他エリア名は正しいエリア名に直す）
・faq-item の数
  → .txt の subtitle_5_n / description_5_n の件数に合わせて調整
・各 #subtitle_5_n / #description_5_n
  → .txt の subtitle_5_n（施設名）、description_5_n（住所・TEL・URL）。リンクは target="_blank" rel="noopener noreferrer" を付与
・#description_5_4 の注意文
  → そのまままたは .txt に合わせる
