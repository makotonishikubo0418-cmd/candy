# ページ作成用

更新日: 2026-07-14

## 目的

ページ制作のたびに、実ファイルから確認できた生成ルール、判断基準、例外、検証結果を追記し、次の制作へ再利用する。

この資料は制作中に成長させる運用メモである。最上位ルールはルート `AGENTS.md` と `HP/AGENTS.md`、正式仕様は `HP/codex/docs/CANDY_MASTER_DOC_INDEX.md` が案内する正本資料を優先する。

## 共通運用

- ページ作成指示を受けたら、既存資料と実ファイルから判断できる内容について確認質問だけを返さず、制作と検証を進める。
- 対象テキストに明記された正しい情報を最優先で使用する。
- 明らかな誤記、未入力、既存ファイルとの矛盾は、完成例と同カテゴリの実データを調査して扱う。
- 推測だけで確定できない重大事項、DB、認証、削除、移動、noindex/index変更は停止条件とする。通常ページの「アップしろ」は本番操作の明示許可として扱う。
- 制作で新しく判明した再利用可能な判断基準を、このファイルへ追記する。

## 報告方法

- 「アップしろ」は、今回の進行内容、関連 `.md` の整合、Commit、Push、自動本番Actions、本番HTTP確認までの一括命令とする。
- 正常時は途中質問を返さず、GitHub APIでActionsを追跡して最後まで進める。
- 通常の1ページ公開は、制作完了後の「アップしろ」から本番URL報告まで5分以内を運用目標とする。
- GitHub Actionsの通常操作にブラウザUIを使用しない。

- Pushを報告するときは、対象CommitのGitHub確認用URLを同時に記載する。
- Actionsを報告するときは、対象Runの確認用URLを同時に記載する。
- 本番反映・ページ公開を報告するときは、作成した全ページの本番確認用URLを同時に記載する。
- 複数ページを代表URL一件で済ませない。
- ローカル作成のみの場合は `確認用URL: 未取得（本番未反映）` と記載する。
- URLは実在と対象を確認してから記載し、未確認URLを推測で作らない。

## areaページ生成

### 基本方式

- `HP/source/template_kagoshima-deliveryhealth-area.html` を丸ごと複製し、対象slugのsource HTMLを作る。
- HTML構造をゼロから作り直さない。
- 公開PHPとページ別dataset PHPは、現行の完成済みareaページと同じ形式を複製する。
- 対象テキストのtitle、description、canonical、画像、本文、店舗、地図、ホテル、周辺スポットを反映する。
- scene、subtitle、description、FAQ、JSON-LDは、反映後の本文件数と順序へ同期する。

### 人気デリヘル店

- 対象テキストに店舗指定がある場合は、その指定を優先する。
- 店舗指定がない場合は、既存ページ同士で同じ組み合わせが続かないように店舗を組み合わせる。
- 店舗ブロックは `HP/source/template_shop.html` の該当店舗を使用する。
- 本文と店舗一覧JSON-LDの店舗名、順序、URL、電話番号、説明を一致させる。

### 移動時間・交通費

- 対象テキストに移動時間・交通費が明記されている場合は、その値を優先する。
- 未記載または明らかな誤記がある場合だけ、対象地域に近い完成済みareaページと同店舗の設定を調査する。
- 単一ページの値をそのままコピーせず、距離帯と既存設定の傾向を比較して対象地域の値を決める。
- 調査で判明した店舗別・距離帯別の具体的な算定傾向は、ページ制作ごとにこの節へ追記する。

### 制作後の確認

- 公開PHP、source HTML、ページ別dataset PHPの3ファイル
- `dataset_base.php` のdataset振り分けとHTMLからPHPへの変換
- `source/area.html` の一覧本文とJSON-LD
- 内部リンクと `sitemap.xml`
- canonical、OGP、h1、パンくず、画像slug
- scene、subtitle、description、FAQの採番
- 本文とJSON-LDの店舗数・順序
- placeholder、旧slug、重複ID
- 画像2枚の存在と読込
- PHP構文、JSON構文、差分検査
